# Assignment 01 — White Blood Cell Image Analysis

**ML4Health 2026**

A single end-to-end assignment over the WBC microscopy dataset, covering three
canonical medical-imaging tasks:

1. **Classification** — predict the WBC type of a single cell.
2. **Segmentation** — assign every pixel to `background`, `cytoplasm`, or `nucleus`.
3. **Multiple Instance Learning (MIL)** — screen synthetic "slide bags" of cells for the rare eosinophil.

All three sections share the same dataset (`segmentation_WBC/Dataset 1`), the
same train / val / test discipline, and the same recurring lesson — class
imbalance and the role of pretrained features.

---

## Prerequisites

- **git** — `git --version`
- **GitHub account** with access to this assignment via GitHub Classroom
- **miniconda** (recommended for managing Python environments)
- **A CUDA GPU** is helpful but not required — every cell runs on CPU too

## 1. Set up your environment

Once per machine:

```bash
conda create -n ml4health python=3.11
conda activate ml4health
pip install -r requirements.txt
```

In every new shell:

```bash
conda activate ml4health
```

## 2. Clone the WBC dataset

The dataset is **not** committed to this repo. Clone it from its upstream
location into the project root:

```bash
git clone https://github.com/zxaoyou/segmentation_WBC.git
```

After cloning you should have `segmentation_WBC/Dataset 1/` containing the
`.bmp` images and matching `.png` masks, plus the per-image
`Class Labels of Dataset 1.csv` file. The notebook expects exactly this layout.

## 3. Open the notebook

```bash
marimo edit notebook.py
```

This is the **only** notebook — it contains all three sections. Read the
markdown headers; run cells top-to-bottom.

## 4. Implement the TODOs

As you reach each section, fill in the functions marked `TODO` in the matching
module:

- **Section A** → [`classification.py`](classification.py) — cell-type classifier, evaluation metrics, MCC.
- **Section B** → [`segmentation.py`](segmentation.py) — 3-class semantic segmentation + Otsu baseline.
- **Section C** → [`mil.py`](mil.py) — mean / max / attention pooling for MIL.

Each function's docstring describes the inputs, outputs, and expected
behaviour. **Don't change function signatures** — the autograder calls them
by name.

## 5. Answer the multiple-choice and numeric questions

Section A also has multiple-choice and short-numeric questions. Use the radio
and number widgets in the notebook itself. A hidden cell at the bottom
auto-saves every widget value to `submission.json` whenever a value changes —
that's the file the autograder reads, so **make sure you've run all cells at
least once before submitting**, otherwise the file won't be regenerated.

## 6. Self-check before submitting

Run the test suite locally:

```bash
pytest tests/ -v
```

This checks both:

- the code you implemented in the three task modules, and
- the answers you typed into the widgets (via `submission.json`).

Some tests require the dataset and / or pretrained weights — those will run on
your machine but skip on the autograder if anything is missing. A green local
run is the strongest signal that you're ready.

## 7. Submit

**Push to `main`** when you are ready:

```bash
git add -A
git commit -m "final submission"
git push origin main
```

Use a separate branch (e.g. `wip`) to save work in progress; only push to
`main` for your final submission. Each push to `main` triggers the autograder.
